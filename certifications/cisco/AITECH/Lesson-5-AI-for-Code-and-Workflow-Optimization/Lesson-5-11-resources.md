# Lesson 5-11: Using Claude Code

> Student follow-along resources, key concepts, and references for this sublesson.

## Overview

**Claude Code** is Anthropic’s **terminal-first** coding agent: it navigates repositories, proposes edits, and may execute commands **after explicit user approval**. This sublesson emphasizes **permission discipline**, **checkpointing** long tasks, and the same verification habits from Lesson 5-6—tests plus human diff review.

## Learning objectives

By the end of this sublesson you should be able to:

- Describe the default interaction loop: propose → approve → execute → verify.
- Explain why “read every suggested shell command” is a security habit, not pedantry.
- Break large tasks into milestones with commits or branches for rollback.
- Pair Claude Code with CI and PR review without bypassing team gates.

## Key concepts

### 1. Permissions as design

Terminal agents sit close to **git, package managers, and environment variables**. Approvals are a **trust boundary**—treat suspicious package installs, `curl | sh`, or mass `rm` patterns like red flags.

### 2. Milestones on long runs

For multi-hour refactors: require intermediate commits, run tests at each milestone, and keep a **human-readable log** of decisions for reviewers.

### 3. Prompting invariants

State **invariants** explicitly: “tests X and Y must pass,” “do not change public API signatures,” “no new dependencies.” Invariants reduce agent drift better than vibes-only prompts.

## Why it matters / What's next

Claude Code is a reference implementation of **gated autonomy**—the same pattern you will see in agentic systems (Lesson 6) at larger scale. **Lesson 5-12** applies the mindset to **secure code review** pipelines.

## Glossary

- **Claude Code** — Anthropic’s agentic coding tool oriented around terminal workflows.
- **Checkpoint** — A deliberate save point (commit/branch/tag) mid-task to bound rollback.
- **Gated execution** — Running shell or write actions only after explicit user approval.

## Quick self-check

1. What should you verify before approving a suggested `npm install` or `pip install`?
2. Why are milestones more important for agents than for one-shot chat?
3. Which tests belong in an “invariant” clause for a risky refactor?
